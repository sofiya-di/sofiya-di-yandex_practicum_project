 **Задачи проекта:**
 
1. Построить предположительную воронку продаж
2. Исследовать путь пользователей до покупки
3. Поиск точек наибольших потерь пользваотелей
4. Анализ результатов А/А/В-тестирования
5. Рекомендации по снижению количества потерь пользователей
 
**Данные:**

данные пользовательской активности

**Инструменты:** 

Python, Pandas, Matplotlib, Seaborn, Plotly, событийная аналитика, продуктовые метрики, визуализация данных, A/B-тестирование, проверка статистических гипотез

**Выводы:**
 
1. Составлен предположительный порядок событий воронки
2. Составлена воронка с долями "идущих пользователей"
3. Выявлена точка наибольших потерь пользователей
4. Проведена проверка на наличие статистической разницы между тестовыми группами для решения вопроса об изменении шрифта
5. Даны рекомендации:
- статистически не было оснований считать доли перехода на страницы в группе со старым шрифтом и измененным шрифтом разными
- рекомендуется не изменять шрифт в приложении (учитывая отсуствие разницы в долях перехода в контрольной и экспериментальной группе)
- рекомендуется проанализировать главный экран ( больше всего пользователей мы теряем на этапе перехода с главной старницы на старницу предложения)
